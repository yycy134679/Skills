# 策略：宠物

## 类目任务

- 核心成交逻辑：同时打动“宠物愿不愿意用/玩”和“主人省不省心”，双重满足才更容易转化。
- 更容易拿曝光的开场：优先给宠物真实反应，或给养宠家庭熟悉的麻烦场景。
- 最容易促成转化的证据：宠物参与度、耐用度、清洁难度、尺寸适配、主人省时省心的变化。

## 曝光优先切入

- 第一秒尽量让宠物出镜，而不是只拍产品。
- 用“这就是我家猫/狗每天最真实的状态”开场，更容易建立可信感。
- 如果是清洁、喂食、饮水类产品，可先给麻烦场景或变化结果。

## 转化优先证据

- 拍宠物第一次接触和持续使用的真实反应。
- 展示尺寸、材质、稳固度、噪音或清洁方式。
- 说明适合的宠物体型、性格或使用场景。
- 回答主人最关心的“会不会闲置、会不会难清理、会不会经不起折腾”。

## 达人拍法要求

- 人设更像“真实养宠家庭分享好用物”，不要像训宠专家。
- 画面以宠物动态、中近景反应和主人操作切换，减少全程口播。
- 节奏上先展示宠物是否愿意接受，再补产品细节。
- 口播要坦诚，例如“我家是偏活泼型/偏谨慎型”，不要替所有宠物下结论。

## 最佳 Hook 模式

- “我家这只本来挺挑的，结果它对这个反应比我预想更快。”
- “如果你每天都在处理这个养宠小麻烦，这个设计会很省心。”
- “我留这个宠物用品，不是因为新鲜，是因为它真的减少了我每天重复做的事。”

## 推荐分镜推进

1. 用宠物反应或养宠痛点开场。
2. 展示产品形态和与宠物的第一次接触。
3. 展示持续互动、使用或进食/饮水过程。
4. 补材质、稳定性、清洁方式、收纳细节。
5. 说明适配的宠物类型或家庭场景。
6. 回答耐用、清洁、接受度等疑问。
7. 用“适合哪类养宠家庭先试”做软性收尾。

## 常见异议与回应

| 异议 | 回应模式 |
|---|---|
| “我家宠物会喜欢吗？” | “只描述当前宠物的真实反应，并补充适配性判断。” |
| “会不会很快玩腻/不用？” | “展示持续使用一段时间的状态或高频场景。” |
| “难清洗吗？” | “把拆洗、擦拭、换水、收纳拍出来。” |
| “经不经得住折腾？” | “拍材质、固定方式和真实互动强度。” |

## 对比规则

```yaml
comparison_recommended: true
comparison_when:
  - 旧方案存在明显的清洁、喂养、互动或收纳麻烦
  - 宠物参与度或主人省心程度可被真实拍到
comparison_type:
  - 旧养宠流程_vs_新流程
  - 无产品状态_vs_使用产品后
```

## 软性 CTA 模式

- “如果你家宠物和这个性格/体型接近，可以先从这个场景试试看。”
- “先看它更适合你家哪种养宠问题，再决定要不要入。”
- “先收藏这条，等你下次补宠物用品时对照会更快。”

## 建议 / 避免

| 建议 | 避免 |
|---|---|
| 宠物真实反应优先于主人夸赞。 | 强行让宠物配合拍摄。 |
| 说明体型、性格和使用场景。 | 说“所有猫狗都会爱”。 |
| 拍主人省心的那一步变化。 | 忽略清洁和耐用问题。 |

## 类目专属合规提醒

- 不要诱导危险互动或让宠物处于明显压力环境。
- 不要承诺所有宠物都适用、都会喜欢、都会立刻适应。
