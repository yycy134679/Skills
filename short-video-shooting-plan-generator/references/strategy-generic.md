# 策略：通用兜底

## 类目任务

- 当类目映射不明确时，提供保守但依然可执行的脚本，让达人至少能拍出“问题 - 演示 - 结果 - 软转化”的基本成交路径。
- 更容易拿曝光的开场：从具体使用问题、结果预告或快速上手动作切入。
- 最容易促成转化的证据：安装/开箱/使用过程、可观察结果、适用场景、是否容易坚持使用。

## 曝光优先切入

- 用最容易被理解的使用场景开场，不要硬猜专业话术。
- 如果产品有明显动作或结果，优先拍动作和结果，不要先拍静物。
- 前 3 秒先解决“这条视频跟我有什么关系”。

## 转化优先证据

- 至少给出一次完整使用动作和一次可观察结果。
- 说明谁最适合、什么场景最适合，避免大而泛。
- 如果有安装、准备或清洁步骤，要让用户感觉门槛可控。
- 避免杜撰参数、功效和行业结论。

## 达人拍法要求

- 人设以“真实使用者分享一个解决方案”为主。
- 画面尽量包含手部操作、产品结果和使用场景，而不只是对镜口播。
- 节奏上用一条最短路径讲明白：为什么看、怎么用、有什么变化。
- 口播坚持务实，不故作权威。

## 最佳 Hook 模式

- “如果你在这个类目里犹豫选哪款，先看这个快速测试。”
- “用一个简单方法，先判断它是否适合你的日常。”
- “不吹不黑，30 秒看完实用演示。”

## 推荐分镜推进

1. 介绍产品和一个常见使用场景。
2. 清晰展示准备/安装步骤。
3. 展示一次直接使用过程。
4. 展示可观察结果。
5. 保守回应一个高频疑问。
6. 用软性建议收尾。

## 常见异议与回应

| 异议 | 回应模式 |
|---|---|
| “这对我有用吗？” | “说明适用场景，并建议小范围试用。” |
| “值得尝试吗？” | “强调使用频率与上手难度，避免强承诺。” |

## 对比规则

```yaml
comparison_recommended: false
comparison_when:
  - 仅在差异可被真实观察时使用
comparison_type:
  - 可选_基线对比
```

## 软性 CTA 模式

- “如果你的需求和这个场景相似，可以先试一次。”
- “先在一个真实场景里测试，再决定。”
- “收藏这份检查清单，和你现在的方案对比。”

## 建议 / 避免

| 建议 | 避免 |
|---|---|
| 语言保持务实、可证据化。 | 在无证据情况下做类目特定承诺。 |
| 明确说明当前使用的是兜底策略。 | 类目不匹配时假装高置信专业判断。 |
| 引导小步试错动作。 | 用高压成交话术收尾。 |

## 类目专属合规提醒

- 不确定的地方宁可保守，也不要冒充专业结论。
- 必须在输出中明确当前使用的是兜底策略。
