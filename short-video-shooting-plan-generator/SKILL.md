# 短视频拍摄方案生成器

## 目标

为短视频带货场景同时生成两份互相关联、但服务对象不同的内容：

1. 给达人直接执行的结构化拍摄脚本。
2. 给 Seedance 直接复制使用的视频生成提示词。

这两份内容共享同一商品策略，但不能互相代替：

1. 达人脚本回答“这条视频怎么拍、怎么讲、怎么转化”。
2. Seedance prompt 回答“这 15 秒连续视频该如何生成”。

达人脚本不是泛泛创意清单，而是要帮助达人完成三件事：

1. 前 3 秒先抓停留，让用户愿意继续看。
2. 中段快速建立信任，用真实证据替代空泛夸赞。
3. 结尾降低决策阻力，用软性 CTA 推动收藏、尝试或下单。

默认全中文输出；达人口播文案使用中英双语对照（中文在上，英文在下）。Seedance prompt 本体按参考规范使用中文字段 + 英文旁白，并固定放在整份结果的最后。

## 输入与默认值

- 提取输入：
  - `product_name: string`
  - `category_raw: string`
  - 若用户给出，也尽量提取：`category_secondary_raw`
  - 若用户给出，也尽量提取：`target_user`、`price_band`、`key_selling_points`、`usage_scene`、`content_constraints`
  - 若用户给出，也尽量提取：`product_image_reference`、`product_visual_details`
- 提取优先级：
  - 优先读取显式字段（如 `Category: ...`、`category is ...`、`类目: ...`）。
  - 仅在没有显式字段时，从自然语言描述中推断类目。
- 固定默认：
  - 平台：TikTok Shop
  - 语言：中文（达人脚本口播列中英双语）
  - 时长：30-45 秒
  - 镜头数：6-8
  - Hook 数量：2（主推 + 备选）
  - CTA 风格：软性引导
  - Seedance 输出：默认必须追加
  - Seedance 结构：15 秒、7 个 Shot、连续视频、竖屏 9:16
  - 未知类目：兜底 `generic`

如果用户没有提供商品图，也必须照常输出 Seedance prompt；此时只能根据商品名与用户显式给出的信息做保守描述，不能因为缺图而跳过这一模块。

## 读取策略文件

按以下顺序读取并只加载必要文件：
1. 读取 `references/taxonomy.md`，将 `category_raw` 归一化为标准类目。
2. 根据标准类目读取对应 `references/strategy-*.md`。
3. 若 `category_normalized=pet-supplies` 且命中二级类目，再追加读取 `references/pet-supplies/<subcategory_normalized>.md` 作为差异层。
4. 若无法命中任何类目，改读 `references/strategy-generic.md`。
5. 最后读取 `references/compliance.md`，执行风险表达替换与提示。
6. 生成最终结果前，读取 `references/seedance-video-prompt.md`，生成独立的 Seedance 视频生成提示词。

二级差异层只负责补充或覆盖一级宠物策略里的差异点，不要把整套一级逻辑复制一遍。若宠物二级无法稳定命中，则只使用一级宠物策略，不要强行猜测。

## 核心脚本原则

- 让每个镜头都承担明确任务，不要重复同一卖点。
- 先展示“用户为什么要停下来看”，再展示“为什么值得信”，最后才是“为什么现在可以试”。
- 优先拍“可观察证据”，例如前后差异、使用过程、材质细节、尺寸参照、真实反应、时间节省、场景适配。
- 避免只堆参数、成分、空泛赞美；必须把卖点翻译成画面动作和口播表达。
- 对比镜头只在差异可真实拍出时使用，不能为了转化硬造对比。
- 如果用户没有提供太多商品信息，基于类目策略做合理假设，但口吻保持克制，不伪造数据、评价或功效。
- 达人脚本与 Seedance prompt 不能互相改写或逐行翻译；它们只能共享目标人群、核心痛点、主打卖点、主证据类型与整体氛围。
- Seedance prompt 不是第二份 Markdown 分镜表，而是按 `references/seedance-video-prompt.md` 生成的一条完整视频生成提示词。
- 若用户提供了商品图或明确外观信息，Seedance prompt 中的商品描述只能基于这些可见信息。
- 若用户没有提供商品图，仍然要输出 Seedance prompt，但商品描述必须保守，避免杜撰精确颜色、配件、材料、结构细节。
- 无图场景下，不要在最终结果里向用户索要商品图，也不要输出“无法保证外观一致性”之类的解释性提醒。

## 生成流程

1. 提取 `product_name` 与 `category_raw`，并尽量收集 `product_image_reference` 或 `product_visual_details`。  
2. 归一化类目，生成：
   - `category_normalized`
   - `strategy_file`
   - `subcategory_normalized`（仅 `pet-supplies` 时可选）
   - `subcategory_label`（仅 `pet-supplies` 时可选）
   - `subcategory_file`（仅 `pet-supplies` 时可选）
3. 从一级策略文件与可选二级差异文件提取：
   - `类目任务`
   - `曝光优先切入`
   - `转化优先证据`
   - `达人拍法要求`
   - `最佳 Hook 模式`
   - `推荐分镜推进`
   - `常见异议与回应`
   - `Comparison Rule`
   - `软性 CTA 模式`
   - `建议 / 避免`
   - `类目专属合规提醒`
   - 若一级与二级差异层存在同名条目，优先采用二级差异层；一级未覆盖的部分继续沿用一级策略。
4. 基于商品和类目，先确定本条视频的：
   - 目标人群
   - 核心痛点
   - 单条视频主打卖点（只选 1-2 个）
   - 主证据类型（例如上脸、上身、对比、试吃、安装、前后变化、宠物反应等）
   - 建议达人出镜方式
5. 生成 2 个 Hook（主推 + 备选）。  
6. 生成 6-8 个分镜，保证总时长在 30-45 秒，并遵守以下推进逻辑：
   - 镜头 1-2：停留段，问题/结果/反差前置
   - 镜头 3-5：证据段，演示主卖点和使用结果
   - 镜头 6-7：异议处理段，回答“适不适合我”“值不值”“难不难用”
   - 最后 1 镜头：软性转化段，降低决策阻力
7. 若 `comparison_recommended: true`，或用户明确要求对比，且差异能被真实观察，则插入至少 1 个对比镜头；否则不强制。  
8. 每个镜头都生成口播文案，且必须是两行：
   - 第一行：中文
   - 第二行：英文
9. Hook 与分镜要尽量贴近达人真实表达，避免“品牌官号腔”或过度营销化措辞。  
10. 使用软性 CTA，不使用高压成交语气。  
11. 应用合规规则：替换达人脚本中的高风险词，保留必要风险提示。  
12. 读取 `references/seedance-video-prompt.md`，基于同一商品策略生成 1 条独立的 Seedance prompt，并遵守以下规则：
   - 固定为 15 秒、7 个 Shot、连续视频、竖屏 9:16。
   - 必须从 Seedance 参考中的 Hook 库中选择 1 种具体 hook 类型。
   - 必须使用参考文件规定的字段顺序生成完整 prompt，而不是改写达人分镜表。
   - 旁白必须是一段连续英文，带说话人特征标注，并在 Shot 1 开始说话。
   - 可复用类目策略中的“目标人群、痛点、卖点、证据类型、达人气质”，但不要把达人脚本的 6-8 镜头逐行映射成 7 个 Shot。
   - 若有商品图，只能描述图中可见的商品信息；若无商品图，只能基于商品名和显式已知信息做保守描述，且不能提示用户“缺图”。
13. 对 Seedance prompt 也应用合规与类目特殊规则；如 Seedance 参考与合规护栏冲突，以更严格的规则为准。
14. 按固定 Markdown 结构输出，并在开头先给出“一句话总结”。若走 `generic`，先附加兜底说明，再在最后输出 Seedance 模块。

## 示例（用于对齐输出预期）

**示例 1**  
输入：`Product: Niacinamide Brightening Serum; Category: skincare`  
输出：中文 TikTok Shop 分镜方案，命中 `beauty-personal-care`，包含目标人群、核心痛点、主打证据、2 个 Hook、6-8 镜头表格、软性转化、合规说明，以及放在最后的 1 条 Seedance 视频生成提示词；达人口播列为中英双语上下两行。

**示例 2**  
输入：`Product: Cat Litter Box; Category: cat litter`
输出：命中 `pet-supplies` 类目，并进一步命中宠物二级差异层 `猫狗如厕用品`；方案应突出清洁流程、异味顾虑、猫咪接受度与主人维护成本，同时在最后追加 1 条宠物场景的 Seedance prompt。若输入为无法识别的一般用品，则命中兜底策略并附加：`因未匹配到具体类目，已使用通用兜底策略。`

## 输出结构（必须完整）

严格输出下列标题，且顺序一致：

1. `一句话总结`
2. `视频简报`
3. `Hook 选项（2个）`
4. `分镜清单（6-8镜头）`（Markdown 表格）
5. `转化设计（软性引导）`
6. `合规说明`
7. `兜底说明`（仅当 `category_normalized=generic` 时输出）
8. `Seedance 视频生成提示词`

`Seedance 视频生成提示词` 必须始终放在整份结果的最后。

### 分镜清单表格列（七要素，禁止缺失）

| 镜头 | 时长 | 画面动作 | 机位/景别 | 口播文案（上中文/下英文） | 卖点传达 | 转化动作 |
|---|---|---|---|---|---|---|

口播文案列必须使用同一个单元格内的双行格式，推荐使用 `<br>` 换行。

### Seedance 视频生成提示词模块格式

- 模块标题固定为 `Seedance 视频生成提示词`。
- 标题下只放 1 段可直接复制的完整提示词，推荐用 fenced code block 包裹，便于复制。
- 模块内部禁止再输出解释、拆解、注意事项或第二份表格。
- 提示词字段顺序必须与 `references/seedance-video-prompt.md` 一致：
  - `[类型]`
  - `[概述]`
  - `[风格]`
  - `[环境]`
  - `[人物与情境]`
  - `[镜头]`
  - `[灯光]`
  - `[产品一致性]`
  - `[背景音]`
  - `[转场/剪辑]`
  - `[动作与旁白]`
- `动作与旁白` 内必须写满 Shot 1 到 Shot 7，并在末尾给出一段连续英文旁白。
- 若无商品图，`[产品一致性]` 不能写成“与提供图片完全一致”，而要改为与“商品名称和已知信息一致、前后一致”的保守表达。

## 视频简报必须包含的信息

`视频简报` 至少包含以下条目：

- 产品
- 类目（原始）
- 类目（归一化）
- 类目（二级）（仅 `pet-supplies` 命中时输出）
- 目标人群
- 核心痛点
- 主打卖点
- 主打成交证据
- 建议达人拍法
- 平台
- 时长目标
- 风格

## 质量约束

- 除口播文案中的英文对照行外，其余内容必须为中文。
- 镜头数必须在 6-8。
- 总时长必须在 30-45 秒。
- Hook 必须为 2 个。
- 每个镜头必须含完整七要素。
- 口播文案必须严格双语两行（中文在上，英文在下）。
- 至少 1 个 Hook 必须直接点出痛点、场景或结果，不能两条都空泛。
- 至少 3 个镜头用于展示“可观察证据”，不能全靠口播硬讲。
- 每个镜头的卖点传达应逐步推进，避免 6-8 个镜头反复重复同一结论。
- 禁止出现绝对化、医疗化、无法证明的承诺表达。
- 对比镜头只能按策略条件触发，不得无条件强插。
- CTA 必须是软性引导（鼓励尝试、强调适配人群与体验）。
- 方案要体现“达人怎么拍”，而不只是“产品卖点是什么”。
- 必须始终输出 `Seedance 视频生成提示词` 模块，且放在整份结果最后。
- Seedance 模块只能有 1 条完整 prompt，不能拆成说明文、字段解释或第二份达人脚本。
- Seedance prompt 必须固定为 15 秒、7 个 Shot、竖屏 9:16。
- Seedance prompt 必须使用中文字段 + 英文旁白；旁白中不能混入中文。
- Seedance prompt 必须与达人脚本逻辑分离，不能把分镜表逐行翻译成 7 个 Shot。
- 若没有商品图，也必须输出 Seedance prompt，且不能向用户索图、不能省略该模块、不能输出“无法保证外观一致性”。
- 无图场景下的商品描述必须保守，不能虚构精确颜色、配件、隐藏结构或包装细节。
- 若 `pet-supplies` 命中二级差异层，Hook、证据镜头、异议回应与 CTA 必须体现该二级方向，而不是退化成泛宠物模板。

## 干跑检查清单

生成前后都执行以下检查：
1. 是否成功归一化类目；若失败是否切换到 `generic`。  
1.1 若 `category_normalized=pet-supplies`，是否正确判断并应用了二级差异层；若不确定是否保持为空而非硬猜。
2. 是否先明确了目标人群、核心痛点、主打卖点与主打证据。  
3. 是否存在 2 个 Hook。  
4. 是否严格为 6-8 个镜头。  
5. 每个镜头是否七要素齐全。  
6. 总时长是否在 30-45 秒。  
7. 是否至少有 3 个镜头在展示真实证据。  
8. CTA 是否保持软性表达。  
9. 合规替换是否完成，并在“合规说明”标记。  
10. 口播文案是否每镜头都为中英双语两行。  
11. 开头是否有“一句话总结”。
12. 若走 `generic`，是否输出“兜底说明”。
13. 是否在最后输出 `Seedance 视频生成提示词` 模块。
14. Seedance prompt 是否固定为 15 秒、7 个 Shot、竖屏 9:16。
15. Seedance prompt 是否是一条完整 prompt，而不是第二份分镜表或说明文。
16. 若无商品图，是否仍输出 Seedance prompt，且没有向用户索图或补充缺图免责声明。
