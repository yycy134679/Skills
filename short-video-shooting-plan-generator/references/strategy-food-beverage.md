# 策略：食品与饮品

## 类目任务

- 核心成交逻辑：先让用户感受到“好吃/好喝/方便/不踩雷”，再补充配料、口感和场景适配。
- 更容易拿曝光的开场：开袋、倒出、拉丝、冒气、第一口反应、冲调变化，往往比产品正面照更抓人。
- 最容易促成转化的证据：真实口感反应、质地状态、成分/配料透明、准备难度、份量感和食用场景。

## 曝光优先切入

- 前 3 秒优先给入口前后的状态变化，而不是先讲品牌故事。
- 如果视觉食欲强，优先拍近景，例如断面、流动感、冒泡、颗粒、冰块碰撞。
- 对饮品或冲调类，优先拍“从无到有”的变化过程。

## 转化优先证据

- 说清味型、甜度、口感层次和熟悉参考物，帮助用户降低踩雷风险。
- 展示准备时间、便携方式和清洗负担，让观众判断是否容易复购。
- 交代食用场景，如下午加餐、通勤、健身后、办公桌、居家囤货。
- 如果有配料或规格优势，用近景点到为止，不要长篇科普。

## 达人拍法要求

- 人设优先真实试吃/试喝，诚实表达比过分夸张的“惊呆了”更可信。
- 画面以食物近景、手部操作、入口反应、桌面场景切换为主。
- 节奏上先给食欲和好奇心，再解释值不值得买。
- 口播要具体描述味道和场景，不要只说“绝了、封神、闭眼入”。

## 最佳 Hook 模式

- “下午 3 点没精神时，我最常拿的就是这个。”
- “这个口味组合我本来不看好，结果意外很搭。”
- “想要省事又不踩雷的加餐，可以先从这个开始。”

## 推荐分镜推进

1. 用开袋、倒出、冲调或第一口反应开场。
2. 展示产品形态和关键口味/配料线索。
3. 给出第一口/第一杯的真实感受。
4. 补充质地、甜度、浓淡、份量感。
5. 展示一个高频食用场景。
6. 回答会不会甜、方便吗、值不值等疑问。
7. 用“适合谁先试”或“先放进采购清单”收尾。

## 常见异议与回应

| 异议 | 回应模式 |
|---|---|
| “会不会太甜？” | “描述风味轮廓，并用熟悉口味做参照。” |
| “方便吗？” | “展示携带、准备时间和清洁负担。” |
| “值不值？” | “说明适用场景与使用频率带来的价值。” |
| “会不会踩雷？” | “用真实反应和具体口感描述替代夸张吹捧。” |

## 对比规则

```yaml
comparison_recommended: false
comparison_when:
  - 仅在口味或便捷性差异足够清晰时使用
comparison_type:
  - 可选_本品_vs_常见替代
```

## 软性 CTA 模式

- “如果这个风味刚好是你喜欢的，可以先试一包。”
- “包里先放一份，看看是否适配你的日常节奏。”
- “先收藏到下次采购清单里。”

## 建议 / 避免

| 建议 | 避免 |
|---|---|
| 展示真实试吃/试饮场景和诚实反应。 | 宣称治病或医疗功效。 |
| 用具体词描述口味。 | 使用“所有人都会爱”这类空泛绝对表达。 |
| 只说可观察事实。 | 承诺身材或体重必然变化。 |

## 类目专属合规提醒

- 不要把普通食品饮料说成功能性治疗或减肥保证方案。
- 口感描述可以具体，但不要替所有人断言“绝不会踩雷”。
