# 策略：家电与厨电

## 类目任务

- 核心成交逻辑：让用户看到“一个原本麻烦的任务，现在更快、更省力、更稳定地完成了”。
- 更容易拿曝光的开场：先给任务结果或痛点场景，不要先讲参数和型号。
- 最容易促成转化的证据：启动门槛、操作步骤、结果质量、噪音/体积、清洁维护、重复使用价值。

## 曝光优先切入

- 先让用户看到“这台机器帮我省掉了什么麻烦”。
- 用结果画面做开场，例如热饮出杯、地面清洁前后、空气状态改善前后。
- 把最能体现效率提升的动作前置，避免前 5 秒都在开箱。

## 转化优先证据

- 连续展示“准备 - 启动 - 输出结果”的完整路径。
- 拍出实际耗时、噪音感受、清洁步骤和收纳占地。
- 给一个每周会重复发生的真实使用场景，证明它不容易吃灰。
- 如果是厨电，拍出成品口感/状态；如果是清洁或环境电器，拍出前后变化和维护难度。

## 达人拍法要求

- 人设优先“真实用户在完成一件日常任务”，而不是“站着读卖点”。
- 画面尽量包含台面、操作手部、成品或结果，不要全程只有产品特写。
- 节奏要让人跟得上操作，关键步骤保留连续镜头。
- 口播重点放在“哪里更省事”“哪里更稳”“哪些人会高频用到”。

## 最佳 Hook 模式

- “如果你每周都要重复做这件事，这台机器能省下来的不是一点点时间。”
- “我买它不是为了多一个电器，是为了把这个最烦的步骤直接省掉。”
- “先别看参数，先看它把这件事做出来到底稳不稳。”

## 推荐分镜推进

1. 用日常任务痛点或结果画面开场。
2. 交代机器体积、放置位置或启动门槛。
3. 连续拍一次完整操作。
4. 展示结果质量、速度或前后差异。
5. 补充噪音、清洁、收纳、耗材等转化细节。
6. 回答“会不会吃灰”“难不难维护”的异议。
7. 用适合谁、从哪个场景开始用做软性收尾。

## 常见异议与回应

| 异议 | 回应模式 |
|---|---|
| “会不会操作复杂？” | “展示从准备到完成的一条连续流程。” |
| “买了会不会闲置？” | “绑定一周至少会发生一次的任务场景。” |
| “清洗维护麻烦吗？” | “直接拍拆洗、擦拭、倒残渣等动作。” |
| “噪音/占地大吗？” | “给空间参照和使用时的实际声音感受描述。” |

## 对比规则

```yaml
comparison_recommended: true
comparison_when:
  - 手工方式或旧设备方式的低效明显
  - 结果质量或耗时差异可被真实拍到
comparison_type:
  - 手动流程_vs_设备流程
  - 旧设备_vs_新设备
```

## 软性 CTA 模式

- “如果这就是你家里高频重复的任务，它会比想象中更实用。”
- “先从你最常用的那个场景开始，看值不值得长期留下。”
- “先收藏这套操作流程，回头实际照着试会更清楚。”

## 建议 / 避免

| 建议 | 避免 |
|---|---|
| 让观众看完整任务被完成。 | 全程讲参数却不出结果。 |
| 拍出维护和收纳成本。 | 假装任何家庭都适合。 |
| 用真实家庭/厨房/清洁场景。 | 宣称零噪音、零故障、永久耐用。 |

## 类目专属合规提醒

- 不要夸大节能、安全、耐久等无法现场证明的结果。
- 需要高温、刀片、电源的场景避免忽略基础安全提示。
