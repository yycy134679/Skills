# 策略：服饰鞋包与配饰

## 类目任务

- 核心成交逻辑：让用户在几秒内代入“穿上之后的自己”，先解决好不好看，再解决合不合适、值不值得买。
- 更容易拿曝光的开场：第一秒就给上身/上脚/背上效果，或直接给穿搭前后反差，少用纯平铺。
- 最容易促成转化的证据：版型、垂感、显比例逻辑、尺码参考、搭配场景、细节做工。

## 曝光优先切入

- 先给“上身效果”，不要先讲面料名词。
- 用“同一个人同一机位”的穿前/穿后反差吸引停留。
- 把最容易出效果的动作放在前 3 秒，例如走动、转身、蹲坐、开合包袋、近景看五金。

## 转化优先证据

- 交代真人身高、体重、试穿尺码或包袋上身比例参照。
- 至少展示正面、侧面、背面中的两个角度。
- 明确展示面料厚薄、弹性、贴肤感、是否透、是否容易皱。
- 用两个真实场景证明“一件多搭”或“通勤/约会/出游”适配。

## 达人拍法要求

- 人设以真实试穿、试背、试搭为主，优先用“我穿/我背起来是什么感觉”。
- 画面以全身中景和局部细节近景交替，避免全程怼脸口播。
- 节奏上前段看效果，中段看细节，后段看适合谁。
- 口播要说结果和感受，例如“腿型看起来更利落”，不要只说“高级感拉满”。

## 最佳 Hook 模式

- “这条真的不是靠滤镜显瘦，是版型一上身就把比例拉出来了。”
- “如果你最怕裤子胯宽、腿短、容易皱，这条的上身状态可以先看一下。”
- “我最近出门最常拿的就是这只包，因为它不是好看而已，是真的顺手。”

## 推荐分镜推进

1. 先给上身/上脚/上肩后的整体效果。
2. 展示最打动人的一个版型或细节变化。
3. 切到侧面/背面，补充轮廓、长度、容量或五金做工。
4. 展示面料、弹性、里衬、拉链、口袋等转化细节。
5. 给出一个日常搭配或使用场景。
6. 回答尺码、身材适配或容量疑问。
7. 用“适合谁”和“先从哪一套搭法开始”做软性收尾。

## 常见异议与回应

| 异议 | 回应模式 |
|---|---|
| “会不会显胖/压个子？” | “用同机位上身效果、站姿和走动镜头解释比例变化。” |
| “尺码准吗？” | “给真人尺码参考，并说明偏宽松/偏修身。” |
| “会不会扎/透/皱？” | “近景拍面料、内里和光下状态，避免空口承诺。” |
| “好不好搭？” | “展示至少一个真实搭配场景，降低想象成本。” |

## 对比规则

```yaml
comparison_recommended: true
comparison_when:
  - 上身前后差异清晰
  - 旧穿搭方案存在明显比例或实用性问题
comparison_type:
  - 穿前_vs_穿后
  - 单穿效果_vs_搭配后效果
```

## 软性 CTA 模式

- “如果你平时最在意上身比例，可以先从这个版型试起。”
- “先照着这个尺码和搭法参考一下，再决定要不要入。”
- “先收藏这套穿法，下次出门照着配会更直观。”

## 建议 / 避免

| 建议 | 避免 |
|---|---|
| 第一秒直接给真人效果。 | 长时间平铺讲设计理念。 |
| 明确尺码、版型和搭配场景。 | 用“谁穿都显瘦”这种绝对表达。 |
| 用真实动作展示服装或包袋状态。 | 只靠滤镜和姿势制造假效果。 |

## 类目专属合规提醒

- 不要承诺“所有身材都适合”“人人穿都显瘦”。
- 塑形、塑身类单品避免夸大身体改变或医疗效果。
