# 策略：家纺布艺

## 类目任务

- 核心成交逻辑：让用户看到“空间更舒服、更整洁、更有质感”，同时把触感、尺寸和打理成本说清楚。
- 更容易拿曝光的开场：先给旧床品、旧窗帘、旧沙发区域的状态，或直接给换上后的氛围反差。
- 最容易促成转化的证据：铺上后的整体观感、面料纹理、厚薄垂感、尺寸适配、打理难度。

## 曝光优先切入

- 第一秒优先给空间焕新后的结果，或者给“旧状态 vs 换新后”的反差。
- 如果面料纹理明显，用近景和自然光拍材质细节。
- 不要只拍折叠状态，尽快让产品回到床、沙发、窗边等真实位置。

## 转化优先证据

- 展示铺、套、挂、盖、整理平整的一整套动作。
- 拍出面料纹理、垂坠感、厚度、遮光或防泼表现等可见结果。
- 给尺寸参照和空间参照，说明适合的床型、窗型或沙发尺寸。
- 回答“会不会起皱”“会不会掉毛”“好不好洗”“值不值得换新”。

## 达人拍法要求

- 人设优先真实居家焕新或卧室整理分享，不要像卖布料参数。
- 画面以铺设过程、材质微距、整体空间和局部触感特写交替。
- 节奏上先给焕新结果，再补材质和打理信息。
- 口播强调“看起来更舒服、摸起来怎样、日常打理麻烦吗”，不要只说高级感。

## 最佳 Hook 模式

- “卧室看起来普通，很多时候差的就是这一层。”
- “我换完它之后最明显的不是好看，是整个空间舒服了很多。”
- “如果你也想低成本把房间氛围拉起来，先看这个前后差别。”

## 推荐分镜推进

1. 用旧空间状态或换新结果开场。
2. 展示产品展开、铺设或悬挂过程。
3. 连续拍一次整理平整、拉开、盖上、触摸的动作。
4. 展示整体空间氛围和材质细节。
5. 补尺寸、厚薄、打理方式或第二场景。
6. 回答起皱、掉毛、清洗和适配问题。
7. 用“适合哪类空间先换”做软性收尾。

## 常见异议与回应

| 异议 | 回应模式 |
|---|---|
| “会不会不好打理？” | “拍清洗、抖平、收纳或日常整理动作。” |
| “尺寸合适吗？” | “给床、窗、沙发的尺寸参照和铺上效果。” |
| “会不会显廉价/没质感？” | “用材质微距和整体空间成片效果回应。” |
| “值不值得换？” | “强调空间氛围和触感改善，而不是只讲参数。” |

## 对比规则

```yaml
comparison_recommended: true
comparison_when:
  - 旧状态与换新后的空间氛围差异明显
  - 面料观感、遮光或整洁度变化可被拍到
comparison_type:
  - 旧空间状态_vs_换新后
  - 铺设前_vs_铺设后
```

## 软性 CTA 模式

- “如果你也想先从一个小区域做焕新，可以先从这类布艺试起。”
- “先看你家的尺寸和风格适不适合，再决定要不要换。”
- “先收藏这条，等下次整理卧室或客厅时直接参考。”

## 建议 / 避免

| 建议 | 避免 |
|---|---|
| 用真实空间拍焕新前后。 | 只拍折叠外观不拍铺设效果。 |
| 说清尺寸、触感和打理成本。 | 承诺绝不过敏、绝不掉毛、绝不起球。 |
| 多拍材质微距和整体氛围。 | 大力拉扯或制造不真实测试。 |

## 类目专属合规提醒

- 不要虚构防螨、防过敏、零掉毛、永久不褪色等结论。
- 功能演示要保守真实，不做极端破坏式测试。
