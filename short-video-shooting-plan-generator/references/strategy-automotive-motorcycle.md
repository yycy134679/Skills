# 策略：汽车与摩托车

## 类目任务

- 核心成交逻辑：围绕“骑行/驾驶更顺手、车内车身更整洁、安装更省心”做拍摄，不卖虚假安全感。
- 更容易拿曝光的开场：先给车内或骑行中的高频麻烦，例如手机没地方放、线材乱、头盔不顺手、车身收纳麻烦、停车后整理费劲。
- 最容易促成转化的证据：安装速度、稳固程度、视线是否顺、空间适配、取放便利、前后变化。

## 曝光优先切入

- 第一秒给真实问题现场，比单拍产品外观更抓人。
- 优先拍“装上后立刻变顺”的结果，例如一放就吸住、一挂就稳、一装就更整齐。
- 如果要拍行驶稳定性，尽量用低风险、可控场景表达，不夸大成安全承诺。

## 转化优先证据

- 连续拍安装过程，证明不是麻烦改装。
- 展示不同角度下的稳固表现或取放便利。
- 给尺寸参照，说明适合的车型位置、车把位置或使用环境。
- 回答“会不会挡视线”“会不会掉”“拆下来麻烦吗”“值不值得留在车上”。

## 达人拍法要求

- 人设优先真实车主或骑行用户分享，不要用过度专业的测评腔。
- 画面以内饰中景、驾驶位视角、手部安装、车把近景、稳固特写为主。
- 节奏上先讲麻烦点，再给安装，再看使用结果。
- 口播要强调顺手、省事、秩序感，不要把小配件说成保命神器。

## 最佳 Hook 模式

- “我车上最烦的不是东西多，是这个位置每次都特别不顺手。”
- “这个配件我留到现在，不是因为新鲜，是因为它真的省事。”
- “如果你也总在开车或骑车时被这个小问题打断，先看它装上去到底顺不顺。”

## 推荐分镜推进

1. 用车内或骑行痛点场景开场。
2. 展示产品形态和安装位置。
3. 连续拍一次安装或卡位过程。
4. 展示使用中的稳固、拿取、收纳或遮挡效果。
5. 补尺寸、材质、兼容位置或清洁细节。
6. 回答挡不挡视线、会不会松、拆洗麻烦吗等疑问。
7. 用“适合哪类车主/骑行人先试”做软性收尾。

## 常见异议与回应

| 异议 | 回应模式 |
|---|---|
| “会不会不稳？” | “用可观察的晃动、固定和取放过程回应，不做绝对承诺。” |
| “安装麻烦吗？” | “拍一步到位或低门槛安装过程。” |
| “挡不挡视线/占不占地方？” | “用第一视角和侧视角展示真实位置。” |
| “适不适合我的车型？” | “说明适合的安装区域和常见使用场景。” |

## 对比规则

```yaml
comparison_recommended: true
comparison_when:
  - 整洁度、取放效率或安装前后差异明显
  - 原有方案存在明显摩擦
comparison_type:
  - 原状态_vs_使用后
  - 原来摆放方式_vs_新方案
```

## 软性 CTA 模式

- “如果你车上也卡在这个位置，可以先从这个小配件试试。”
- “先看你的车型或骑行习惯匹不匹配，再决定要不要入。”
- “先收藏这套装法，回头对照自己的车位会更快判断。”

## 建议 / 避免

| 建议 | 避免 |
|---|---|
| 用真实车内或骑行环境拍使用结果。 | 把配件说成绝对安全装备。 |
| 展示安装和拆卸门槛。 | 虚构所有车型都适用。 |
| 多拍顺手感和整理效果。 | 在高风险驾驶场景夸张演示。 |

## 类目专属合规提醒

- 不要承诺防事故、绝对稳固、完全不挡视线或零风险使用。
- 行车相关镜头避免诱导危险驾驶或分心操作。
