# 策略：电脑

## 类目任务

- 核心成交逻辑：把抽象配置翻译成桌面效率、办公体验或娱乐体验的具体好处，让用户看到“它能把我现在的电脑流程优化在哪里”。
- 更容易拿曝光的开场：从桌面凌乱、接口不够、打字不顺、屏幕视角别扭、会议体验差等办公高频痛点切入。
- 最容易促成转化的证据：连接门槛、兼容状态、桌面布局变化、打字/显示/收音体验、便携或稳定性。

## 曝光优先切入

- 第一秒给真实工位问题，而不是先报型号和参数。
- 让产品和电脑主机、笔记本、显示器、键鼠同框，建立场景关系。
- 如果结果能立刻看到，例如桌面更整齐、屏幕更舒服、打字更顺，优先先给结果。

## 转化优先证据

- 展示插接、摆放、支撑、抬升、切换等低门槛动作。
- 拍出尺寸比例、接口位置、稳定度、键程/触感、显示效果或桌面占用。
- 给出典型工作流或学习场景，说明它不会闲置。
- 回答“兼容吗”“会不会鸡肋”“值不值升级”“占地方吗”。

## 达人拍法要求

- 人设以真实办公、学习或创作用户为主，而不是硬件跑分测评腔。
- 画面以桌面俯拍、侧面支撑、接口近景、打字或使用反馈为主。
- 节奏要快，但关键连接和体验反馈要给连续镜头。
- 口播重点放在“哪一步更顺”“桌面更清楚了什么”“长期用起来更舒服的地方”。

## 最佳 Hook 模式

- “我工位最烦的不是设备少，是每次都卡在这个小问题上。”
- “这个电脑配件最有用的地方，不是参数，是让我每天少折腾很多次。”
- “如果你电脑桌也乱成这样，先看这个改动有没有意义。”

## 推荐分镜推进

1. 用桌面或使用流程中的真实痛点开场。
2. 展示产品形态和与电脑设备的关系。
3. 连续拍一次连接、摆放、安装或切换过程。
4. 展示实际使用后的变化，例如视角、稳定性、收纳或输入体验。
5. 补尺寸、接口、材质、便携或第二场景。
6. 回答兼容、占地、鸡肋与否等疑问。
7. 用“适合哪类工位先升级”做软性收尾。

## 常见异议与回应

| 异议 | 回应模式 |
|---|---|
| “兼容我的设备吗？” | “明确适配对象或接口，并拍出连接状态。” |
| “会不会很鸡肋？” | “绑定高频工位场景，说明每天会怎么用。” |
| “占地方吗？” | “给桌面比例和收纳状态参照。” |
| “值不值得换？” | “强调长期使用体验和桌面效率提升，而不是堆参数。” |

## 对比规则

```yaml
comparison_recommended: true
comparison_when:
  - 桌面效率、输入体验或视角舒适度差异清晰
  - 原方案存在明显摩擦
comparison_type:
  - 原工位状态_vs_优化后
  - 原设备使用方式_vs_配件介入后
```

## 软性 CTA 模式

- “如果这正好卡在你的工位流程里，它会是很实用的小升级。”
- “先看你的设备接口和桌面空间，再决定是不是你需要的那一类配件。”
- “先收藏这套工位思路，回头对照自己的桌面会更快判断。”

## 建议 / 避免

| 建议 | 避免 |
|---|---|
| 把产品放进真实工位中展示。 | 只讲规格和参数。 |
| 让观众看见连接和使用结果。 | 虚构兼容性、刷新率、续航等精确数据。 |
| 重点解释效率与舒适度变化。 | 夸大成专业设备替代品。 |

## 类目专属合规提醒

- 不要凭空承诺兼容性、性能提升、耐久或绝对稳定性。
- 涉及供电、接口、画质时，用保守、可验证的表达。
