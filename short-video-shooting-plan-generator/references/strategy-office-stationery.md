# 策略：办公文具

## 类目任务

- 核心成交逻辑：让用户看到“桌面更顺、书包更清、记录更快、办公学习摩擦更少”。
- 更容易拿曝光的开场：优先拍桌面凌乱、找东西慢、书包翻半天、记笔记不顺这种日常烦点。
- 最容易促成转化的证据：容量分层、取放效率、书写/整理手感、占地大小、便携度和场景适配。

## 曝光优先切入

- 第一秒最好给“乱”和“慢”的状态，再展示产品介入后的顺滑感。
- 文具类优先拍真实使用动作，例如拉链打开、翻页、书写、分类拿取。
- 如果是桌面收纳类，优先拍前后对比；如果是书写工具类，优先拍落笔和手感反馈。

## 转化优先证据

- 用“实物装进去”的方式展示容量和分层，而不是只说能装很多。
- 拍出打开、拿取、归位、携带这些高频动作。
- 说明适合通勤办公、学生党、备考、会议等哪种使用场景。
- 回答“占不占地方”“会不会乱”“耐不耐用”“值不值得换”。

## 达人拍法要求

- 人设更像通勤博主、学生博主或桌面效率博主，而不是纯产品介绍员。
- 画面以俯拍桌面、手部操作、书包/办公桌场景切换为主。
- 节奏上先让观众看到效率问题，再给出整理后的顺手感。
- 口播要具体说“拿东西更快”“分类更清楚”“桌面不挤了”，少用空泛治愈词。

## 最佳 Hook 模式

- “我桌面真正让人烦的不是东西多，是每次都找不到要用的那一个。”
- “如果你包里一乱起来就特别容易心烦，这种分层设计会很有感。”
- “这个文具我会一直用，不是因为好看，是因为真的省时间。”

## 推荐分镜推进

1. 用桌面/书包混乱或使用摩擦开场。
2. 展示产品外观和开合方式。
3. 连续拍一次装入、分类、拿取过程。
4. 展示使用后的整齐度或书写手感。
5. 补尺寸、材质、便携和第二使用场景。
6. 回答容量、占地、耐用或清洁疑问。
7. 用“适合哪类办公/学习人群”做软性收尾。

## 常见异议与回应

| 异议 | 回应模式 |
|---|---|
| “真的能装吗？” | “用实物装入过程和层数展示。” |
| “会不会占地方？” | “给桌面或书包空间参照。” |
| “拿取麻烦吗？” | “拍开合和拿取速度，不靠空口描述。” |
| “值不值换？” | “绑定高频使用场景，强调省掉的时间和整理成本。” |

## 对比规则

```yaml
comparison_recommended: true
comparison_when:
  - 桌面或书包前后秩序变化明显
  - 原有方案存在明显找东西慢、收纳乱的问题
comparison_type:
  - 原桌面/书包状态_vs_整理后
  - 原来文具方案_vs_新方案
```

## 软性 CTA 模式

- “如果你也每天在这个环节浪费时间，可以先从这类整理方案试起。”
- “先看你的桌面或包内场景适不适合，再决定要不要换。”
- “先收藏这个整理思路，回头照着摆会更直观。”

## 建议 / 避免

| 建议 | 避免 |
|---|---|
| 让观众看到真实整理效率。 | 只讲颜值不讲使用逻辑。 |
| 用实物和动作证明容量。 | 保证成绩提升或效率必然变高。 |
| 明确办公/学习使用场景。 | 过度制造焦虑式学习话术。 |

## 类目专属合规提醒

- 不要承诺成绩提升、专注力必然改善、学习效果立刻变好。
- 书写、护眼、坐姿等相关话术避免伪科学或医疗化表达。
