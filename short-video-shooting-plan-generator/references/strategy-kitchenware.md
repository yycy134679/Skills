# 策略：厨房用品

## 类目任务

- 核心成交逻辑：让用户看到“备餐、烹饪、收纳或清洗这件事变得更顺手了”，而不是只夸产品外形。
- 更容易拿曝光的开场：先给切配麻烦、锅具粘锅、台面凌乱、保鲜不方便、餐桌动线不顺等真实厨房摩擦。
- 最容易促成转化的证据：使用动作、受热/盛装状态、容量尺寸、清洗难度、收纳效率和场景适配。

## 曝光优先切入

- 第一秒优先拍“正在做饭时最烦的那一步”，比静态摆拍更能抓停留。
- 如果产品结果能直接看到，例如不粘、好盛、好收、好拿，优先先给结果。
- 让产品进入真实厨房台面或餐桌，而不是停留在白底介绍。

## 转化优先证据

- 连续拍一次真实使用过程，例如切、翻、装、收、清洗。
- 给尺寸参照、容量参照和台面占地参照。
- 展示材质触感、边角处理、收纳叠放或搭配使用方式。
- 回答“好不好洗”“会不会占地方”“够不够实用”“值不值得换”。

## 达人拍法要求

- 人设优先真实下厨或居家餐厨分享，不要像专业厨具导购。
- 画面以台面俯拍、手部操作、出锅/盛装结果和收纳细节为主。
- 节奏上先看使用动作，再补细节和适合谁。
- 口播要说“哪里更顺手”“哪里更省时间”“哪里更整齐”，少讲空泛高级感。

## 最佳 Hook 模式

- “我做饭最烦的不是步骤多，是每次都卡在这一步。”
- “这个厨房小东西不花哨，但真的把台面变顺了。”
- “如果你也总被这件小事打断，先看它到底有没有更省事。”

## 推荐分镜推进

1. 用备餐、烹饪或收纳中的真实痛点开场。
2. 展示产品形态和核心使用位置。
3. 连续拍一次完整使用动作。
4. 展示结果状态，例如装得下、翻得开、收得整齐、清洗更快。
5. 补尺寸、材质、叠放或第二场景。
6. 回答清洗、占地、耐用或适配问题。
7. 用“适合哪类厨房先试”做软性收尾。

## 常见异议与回应

| 异议 | 回应模式 |
|---|---|
| “真的好用吗？” | “用连续动作和结果画面证明，不靠空口夸。” |
| “会不会难洗？” | “直接拍冲洗、擦拭或收纳动作。” |
| “会不会占地方？” | “给台面、抽屉或橱柜空间参照。” |
| “值不值换？” | “绑定高频做饭或备餐场景，强调省掉的麻烦。” |

## 对比规则

```yaml
comparison_recommended: true
comparison_when:
  - 备餐效率、整洁度或使用顺手度差异清晰
  - 原有方案存在明显摩擦
comparison_type:
  - 原流程_vs_使用后流程
  - 台面原状态_vs_整理后
```

## 软性 CTA 模式

- “如果你家厨房也卡在这个步骤，可以先从这类工具试起。”
- “先看你的台面和做饭习惯适不适合，再决定要不要换。”
- “先收藏这条，等下次整理厨房或买餐厨用品时照着看。”

## 建议 / 避免

| 建议 | 避免 |
|---|---|
| 用真实厨房动作证明价值。 | 只拍外观不拍使用结果。 |
| 给出尺寸和容量参照。 | 承诺全材质通用、永不粘、永不变形。 |
| 让观众看见清洗和收纳成本。 | 把普通工具吹成专业厨师级神器。 |

## 类目专属合规提醒

- 不要虚构食品级、耐高温、耐刮、绝对不粘等无法验证的结论。
- 涉及刀具、明火、热油场景时，避免省略基础安全提醒。
