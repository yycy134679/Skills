# 策略：通用兜底

## 类目意图

当类目映射不明确时，提供保守且可执行的脚本。表达要克制，演示要通用。

## 最佳 Hook 模式

- “如果你在这个类目里犹豫选哪款，先看这个快速测试。”
- “用一个简单方法，先判断它是否适合你的日常。”
- “不吹不黑，30 秒看完实用演示。”

## 演示顺序

1. 介绍产品和一个常见使用场景。
2. 清晰展示准备/安装步骤。
3. 展示一次直接使用过程。
4. 展示可观察结果。
5. 保守回应一个高频疑问。
6. 用软性建议收尾。

## 常见异议与回应

| 异议 | 回应模式 |
|---|---|
| “这对我有用吗？” | “说明适用场景，并建议小范围试用。” |
| “值得尝试吗？” | “强调使用频率与上手难度，避免强承诺。” |

## 对比规则

```yaml
comparison_recommended: false
comparison_when:
  - 仅在差异可被真实观察时使用
comparison_type:
  - 可选_基线对比
```

## 软性 CTA 模式

- “如果你的需求和这个场景相似，可以先试一次。”
- “先在一个真实场景里测试，再决定。”
- “收藏这份检查清单，和你现在的方案对比。”

## 建议 / 避免

| 建议 | 避免 |
|---|---|
| 语言保持务实、可证据化。 | 在无证据情况下做类目特定承诺。 |
| 明确说明当前使用的是兜底策略。 | 类目不匹配时假装高置信专业判断。 |
| 引导小步试错动作。 | 用高压成交话术收尾。 |
