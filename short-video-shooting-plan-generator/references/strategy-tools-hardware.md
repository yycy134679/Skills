# 策略：工具五金与家装维修

## 类目任务

- 核心成交逻辑：让用户看到“原本费劲的修理/安装任务，现在更省力、更好上手、更不容易出错”。
- 更容易拿曝光的开场：优先拍一个卡住人的小维修问题，例如拧不动、装不平、找不到工具、步骤太麻烦。
- 最容易促成转化的证据：真实操作过程、力度反馈、收纳完整度、配件齐全度和上手门槛。

## 曝光优先切入

- 第一秒给“需要修/装”的问题现场，而不是先拍工具盒。
- 如果能拍出“原来费劲 vs 现在省力”的差异，优先用对比切入。
- 工具类更适合看动作，不适合纯参数口播开场。

## 转化优先证据

- 连续拍一次完整任务，例如拧、钻、装、量、切。
- 展示常用配件、收纳方式、电量/续航线索或握持细节。
- 说明适合新手的小维修场景，不要一上来就拍太硬核施工。
- 回答“会不会难用”“家里会不会闲置”“够不够力”“配件全不全”。

## 达人拍法要求

- 人设优先生活维修分享或家居改造分享，不要像专业工地教学。
- 画面以手部操作、工件近景、安装前后结果和工具收纳特写为主。
- 节奏上先让人看明白要解决什么问题，再看工具如何解决。
- 口播要讲省力、顺手、适合家用，不要一味强调专业级。

## 最佳 Hook 模式

- “家里这种小修小补最烦的不是难，是每次都卡在第一步。”
- “我留下这套工具，不是因为专业，而是它把很多小问题变得好解决了。”
- “如果你也总被这种安装步骤卡住，先看它到底能省多少力。”

## 推荐分镜推进

1. 用维修/安装问题开场。
2. 展示工具套装内容和核心部件。
3. 连续拍一次实际操作。
4. 展示完成后的结果和手感反馈。
5. 补配件、收纳、电量或第二场景。
6. 回答新手能不能用、值不值、会不会闲置。
7. 用“适合哪类家庭先备一套”做软性收尾。

## 常见异议与回应

| 异议 | 回应模式 |
|---|---|
| “我不会用怎么办？” | “用最基础的家用场景演示操作门槛。” |
| “够不够力？” | “用当前任务的完成效果回应，不编造极限性能。” |
| “会不会买来吃灰？” | “绑定家里会反复遇到的小维修场景。” |
| “配件够用吗？” | “展示常见批头/配件和收纳逻辑。” |

## 对比规则

```yaml
comparison_recommended: true
comparison_when:
  - 手动费力程度或安装效率差异明显
  - 原有做法存在明显摩擦
comparison_type:
  - 原来处理方式_vs_工具介入后
  - 未完成状态_vs_安装完成后
```

## 软性 CTA 模式

- “如果你家里经常会遇到这类小问题，先备一套这种工具会更省心。”
- “先看你最常碰到的维修场景匹不匹配，再决定要不要入。”
- “先收藏这套用法，回头家里要修东西时会很有参考价值。”

## 建议 / 避免

| 建议 | 避免 |
|---|---|
| 用真实任务证明工具价值。 | 只讲扭矩、规格，不拍结果。 |
| 优先拍家用小维修场景。 | 承诺零风险、无限承重、绝不损坏。 |
| 说明新手也能理解的上手路径。 | 把家用工具夸成万能专业设备。 |

## 类目专属合规提醒

- 不要承诺绝对安全、绝不失手、无限耐用或超出证据范围的承重能力。
- 需要切割、钻孔、带电场景时，避免省略必要的基础安全提醒。
